# Chtholly 发展路线图

本文档旨在记录 Chtholly 编程语言的开发进度和未来计划。

## 第一阶段：奠定基础

- [ ] **环境搭建与项目初始化**:
  - [x] 设置 Rust 开发环境。
  - [x] 配置 LLVM 18 依赖。
  - [x] 初始化 Cargo 项目。
  - [x] 集成 `inkwell` crate 并配置。

- [x] **基础语法解析**:
  - [x] 实现词法分析器（Tokenizer/Lexer），能够识别基本 Token（关键字、标识符、操作符、字面量等）。
  - [x] 实现语法分析器（Parser），构建抽象语法树（AST）。
  - [x] 支持解析基础表达式（如整数、浮点数）。
  - [x] 支持解析变量声明（`let`, `mut`）。

- [x] **基础代码生成**:
  - [x] 实现基于 LLVM 的代码生成器。
  - [x] 为基础表达式和变量声明生成 LLVM IR。
  - [x] 实现 `main` 函数作为入口点。
  - [ ] 编译并运行简单的 "Hello, World!" 程序。

- [ ] **类型系统**:
  - [ ] 实现基础类型检查（`i32`, `f64`, `bool`, `char`）。
  - [ ] 支持类型推导与类型注解。

## 第二阶段：核心功能

- [ ] **控制流**:
  - [ ] 实现 `if-else` 表达式。
  - [ ] 实现 `while` 循环。
  - [ ] 实现 `for` 循环。

- [ ] **函数**:
  - [ ] 实现函数定义与调用。
  - [ ] 支持函数参数传递。
  - [ ] 支持函数返回值。

- [ ] **所有权与借用**:
  - [ ] 实现所有权系统（移动语义）。
  - [ ] 实现借用与引用（`&`, `&mut`）。
  - [ ] 在编译期进行所有权和借用规则检查。

## 第三阶段：高级特性

- [ ] **复合数据类型**:
  - [ ] 实现 `struct` 定义与实例化。
  - [ ] 实现 `class` 定义、构造函数和成员函数。
  - [ ] 实现 `enum` 定义与模式匹配。

- [ ] **泛型**:
  - [ ] 实现泛型函数。
  - [ ] 实现泛型 `struct` 和 `class`。

- [ ] **错误处理**:
  - [ ] 实现 `Result<T, E>` 枚举。
  - [ ] 实现 `?` 操作符用于错误传播。

- [ ] **模块系统**:
  - [ ] 实现 `import` 语句。
  - [ ] 支持文件模块和标准库模块。
